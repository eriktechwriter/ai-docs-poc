<tr>
<td><p>\a</p></td>
<td><p>The alert (bell) character (ASCII 0x07)</p></td>
</tr>
<tr>
<td><p>\b</p></td>
<td><ul>
<li><p>If this meta-symbol is within square brackets [] or by itself, it will be treated as the backspace character (ASCII 0x08). For example, [\b] or \b</p></li>
<li><p>If this meta-symbol is at the beginning (or end) of a regular expression, it means any matched string of the regular expression must check whether the left (or right) side of the matched string is a boundary. For example:</p>
<ul>
<li><p>\bluck &gt; left side must be the boundary</p></li>
<li><p>luck\b &gt; right side must be the boundary</p></li>
<li><p>\bluck\b &gt; both sides must be the boundary</p></li>
</ul></li>
<li><p>If this meta-symbol appears in the middle of a regular expression, it will cause a syntax error.</p></li>
</ul></td>
</tr>
<tr>
<td><p>\f</p></td>
<td><p>The form-feed character (ASCII 0x0C)</p></td>
</tr>
<tr>
<td><p>\n</p></td>
<td><p>The newline (line feed) character (ASCII 0x0A)</p></td>
</tr>
<tr>

<td><p>The normal (horizontal) tab character (ASCII 0x09)</p></td>
</tr>
<tr>
<td><p>\v</p></td>
<td><p>The vertical tab character (ASCII 0x0B)</p></td>
</tr>

<td><p>\n</p></td>

:::tip

If your expression includes the characters \\ | ( ) { } \[ \] . ^ $ \* + or ?, you must escape them by using a \\ immediately before the character. Otherwise, they will be assumed to be regular expression operators rather than literal characters.
:::
