# Greedy quantifiers

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr>
<th><p>EXPRESSION</p></th>
<th><p>DESCRIPTION</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>R?</p></td>
<td><p>Matches R, once or not at all</p></td>
</tr>
<tr>
<td><p>R*</p></td>
<td><p>Matches R, zero or more times</p></td>
</tr>
<tr>
<td><p>R+</p></td>
<td><p>Matches R, one or more times</p></td>
</tr>
<tr>
<td><p>R{n}</p></td>
<td><p>Matches R, exactly n times</p></td>
</tr>
<tr>
<td><p>R{n,}</p></td>
<td><p>Matches R, at least n times</p></td>
</tr>
<tr>
<td><p>R{n,m}</p></td>
<td><p>Matches R, at least n but no more than m times</p></td>
</tr>
</tbody>
</table>

R is a regular expression.

Trend Micro does not recommend using ".\*" in a regular expression. ".\*" matches any length of letters and the large number of matches may increase memory usage and affect performance.

For example:

If the content is 123456abc, the regular expression ".\*abc" match results are:

- 12345abc

- 23455abc

- 3456abc

- 456abc

- 56abc

- 6abc

- abc

In this example, replace ".\*abc" with "abc" to prevent excessive use of resources.
:::
