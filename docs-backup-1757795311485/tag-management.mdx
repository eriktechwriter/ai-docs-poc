# Tag Management

:View and manage the tags your organization uses throughout Trend Vision One.

:::warning[Important]

This is a "Pre-release" feature and is not considered an official release. Please review the [Pre-release disclaimer](pre-release-disclaimer.mdx) before using the feature.
:::

:::note

This feature is not available in all regions.
:::

The **Tag Management** app allows you to create and manage tags, create rules to automatically tag your assets, and gain centralized visibility into all the Trend Vision One tags used by your organization. By leveraging asset tagging, you can keep your assets organized and gain additional context.

The following table outlines the actions available in the **Tag Management** app.

<colgroup>
<col style="width: 25%" />
<col style="width: 75%" />
</colgroup>

<th><p>Action</p></th>
<th><p>Description</p></th>
</tr>

<tr>
<td><p>Manage custom tags</p></td>
<td><p>Go to <strong>Tag Management → Tag library → All custom tags</strong> to create and manage custom tags for your organization.</p></td>
</tr>
<tr>
<td><p>Manage customizable platform tags</p></td>
<td><p>Go to <strong>Tag Management → Tag library → Customizable</strong> to edit, delete, and manage customizable platform tags.</p>


:::note

<p>You can add tag values to customizable platform tags and assign the tags to assets.</p>


:::

</td>
</tr>
<tr>
<td><p>Manage assignable platform tags</p></td>
<td><p>Go to <strong>Tag Management → Tag library → Assignable</strong> to edit and manage assignable platform tags.</p>


:::note

<p>Assignable platform tags have tag values defined by Trend Micro, but you can assign the tags to assets.</p>


:::

</td>
</tr>
<tr>
<td><p>Sync and manage external tags</p></td>
<td><p>Go to <strong>Tag Management → External tags</strong> to create cloud mappings and manage your third-party tags in Trend Vision One.</p></td>
</tr>
<tr>
<td><p>Create automated tagging rules</p></td>
<td><p>Go to <strong>Tag Management → Automated tagging</strong> to create and manage automated tagging rules.</p></td>
