# Use the SDK

How to use the Go SDK for Trend Vision One™ File Security

After initializing the Go SDK for File Security, you can start using it to interact with File Security. Here are some basic examples of how to use the SDK.

### Procedure {#procedure}

- Scan a file.

  ```bash
  filePath := "path/to/your/file.txt"
  tags := []string{"tag1", "tag2"}

  response, err := client.ScanFile(filePath, tags)
  if err != nil {
      // Handle scanning error
      panic(err)
  }

  // Use the 'response' as needed
  ```

- Scan a buffer. You can use up to eight tags each with a maximum length of 63 characters.

  ```bash
  filePath := "path/to/your/file.txt"
  tags := []string{"tag1", "tag2"}

  response, err := client.ScanFile(filePath, tags)
  if err != nil {
      // Handle scanning error
      panic(err)
  }

  // Use the 'response' as needed
  ```

## Using advanced functions {#using-advanced-functions}

The Go SDK provides additional functions for advanced use such as enabling PML, dumping the configuration, and cleaning up resources.

### Procedure {#procedure-1}

- Dump the configuration.

  You can dump the current configuration for debugging purposes.

  ```bash
  configDump := client.DumpConfig()
  fmt.Println("SDK Configuration:\n", configDump)
  ```

- Clean up resources.

  You can destroy the SDK client when you are done to release any allocated resources.

  ```bash
  client.Destroy()
  ```

- Enable Predictive Machine Learning (PML)

  ```bash
  client.SetPMLEnable()
  ```

- Enable SPN feedback

  ```bash
  client.SetFeedbackEnable()
  ```

## Viewing Examples {#viewing-examples}

The `tools/` directory of the SDK package contains two important files with code examples.

### Procedure {#procedure-2}

- Refer to these examples for a deeper understanding of how to integrate and use the SDK with File Security.

  - `client.go`: This file contains the main client initialization logic and functions for scanning a single file.

  - `scanfiles.go`: This file provides examples of how to scan multiple files using the SDK.

## Using client tools {#using-client-tools}

- Execute `make build` in the root directory to build the client tools.

  The build process produces the following client tools in the `tools/`directory:

  - `client`

    This program supports servers with Google™ remote procedure call (gRPC).

    ```bash
    client [command-line flags]
    ```
