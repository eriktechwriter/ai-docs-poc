# Log types

Find out the type of information you want to query in logs.

The following table explains the available log types when conducting a search on log data. Every log type includes log facets for granular analysis of log data. For details, see [Log facets](log-facets.mdx).

**Log Type Descriptions**

<table>
<colgroup>
<col style="width: 33%" />
<col style="width: 67%" />
</colgroup>
<thead>
<tr>
<th><p>Log Type</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Detection logs</p></td>
<td><p>Information about email messages, chat messages, and files detected with threats or data leakage, as well as information about files and URLs submitted to Virtual Analyzer for threat analysis in a virtual sandbox.</p>
<p>This log type consolidates the following log types in the old management console: Security Risk Scan, Ransomware, Virtual Analzyer, and Data Loss Prevention</p></td>
</tr>
<tr>
<td><p>Quarantine logs</p></td>
<td><p>Information about email messages and files quarantined due to threats or policy violations.</p></td>
</tr>
<tr>
<td><p>URL click tracking logs</p></td>
<td><p>Information about user clicks on URLs in incoming email messages and the actions taken for the clicked URLs.</p></td>
</tr>
<tr>
<td><p>Email tracking logs</p></td>
<td><p>Information about how the Exchange Online or Gmail email messages are routed to Cloud Email and Collaboration Protection for Inline Protection, including where Cloud Email and Collaboration Protection gets the message and sends the message back to.</p></td>
</tr>
<tr>
<td><p>API integration logs</p></td>
<td><p>Information about the action taken on an email message matching any item in the Blocked Lists for Exchange Online configured through the Threat Remediation API.</p></td>
</tr>
<tr>
<td><p>Audit logs</p></td>
<td><p>Information about user log-on sessions, policy change events, quarantine management operations, and other management events.</p></td>
</tr>
</tbody>
</table>
