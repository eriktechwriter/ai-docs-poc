# Data that will not be migrated

The following table lists the settings on the IMSS or IMSVA management console that will not be migrated to Cloud Email Gateway Protection and describes the reason. All settings on the EUQ management console will not be migrated.

:::note

For details about all the settings that are not migrated, see the data migration report downloaded from the Cloud Email Gateway Protection administrator console when the migration completes.
:::

<tr>
<td><p><strong>Dashboard</strong></p></td>
<td><p>All settings</p></td>
<td><p>The dashboard is a statistical summary of past mail traffic and scanning results. Cloud Email Gateway Protection provides a more powerful dashboard feature.</p></td>

<td><p>IMSS or IMSVA uses the <strong>Internal Addresses</strong> menu to determine mail traffic direction in policy configuration. This is unnecessary in Cloud Email Gateway Protection.</p></td>
</tr>

<td><p><strong>Policy → Smart Protection</strong></p></td>

</tr>

<td><p><strong>Mail Areas &amp; Queues</strong></p></td>

<td><p><strong>Administration</strong></p></td>
