 1.1.17 - Ensure that the controller-manager.conf file permissions are set to 600 or more restrictive (Automated) Profile applicability: Level 1 - Master Node Ensure that the controller-manager.conf file has permissions of 600 or more restrictive. The controller-manager.conf file is the kubeconfig file for the Controller Manager. You should restrict its file permissions to maintain the integrity of the file. The file should be writable by only the administrators on the system. Note By default, controller-manager.conf has permissions of 640. Audit Run the following command (based on the file location on your system) on the Control Plane node. stat -c %a /etc/kubernetes/controller-manager.conf Verify that the permissions are 600 or more restrictive.
