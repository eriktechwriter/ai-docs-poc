## Writing Best Practices

### Headings
- Use proper heading hierarchy (H1 → H2 → H3)
- Don't skip heading levels
- Keep headings descriptive

### Links
- Use descriptive link text: [VS Code documentation](https://code.visualstudio.com/docs)
- Avoid generic text like "click here" or "read more"

### Images
When adding images, always include alt text:
```markdown
![Screenshot of VS Code interface showing the Explorer panel](./screenshot.png)
```

### Lists
- Use bullet points for unordered lists
- Use numbers for sequential steps
- Keep list items parallel in structure

## Quality Checklist

When writing documentation, check for:
- [ ] Clear, descriptive title
- [ ] Proper frontmatter with title, description, tags
- [ ] Logical heading hierarchy
- [ ] Descriptive alt text for images
- [ ] Descriptive link text
- [ ] Spell checking passed
- [ ] Markdown linting passed
- [ ] Content is accessible and inclusive
