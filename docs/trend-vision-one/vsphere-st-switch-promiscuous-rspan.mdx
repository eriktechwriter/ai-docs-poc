# Configuring external network traffic with the VMware vSphere Standard Switch (promiscuous mode/RSPAN)

Configure network settings in VMware vCenter to allow Virtual Network Sensor to monitor data.

:::warning[Important]

The following instructions are valid as of June 2023.
:::

### Procedure {#procedure}

1.  Sign in to the VMware vCenter console using an admin account, and then click the ![](/images/VMware_vCenter_server_icon=GUID-20230329175050.webp) tab.

2.  Right-click the IP address for your server and select **Add Networking...**

    <figure>
    <img src="./images/VMware_Network_1=d247c9e0-2ed8-4b99-8d76-66a4feb9db3d.webp" />
    </figure>

3.  On the **Select connection type** screen, select **Virtual Machine Port Group for a Standard Switch** and click **NEXT**.

    <figure>
    <img src="./images/VMware_Network_2=5750f223-cda2-4cf2-8eca-a9cc92401ba8.webp" />
    </figure>

4.  On the **Select target device** screen, select **New standard switch** and specify `1600` for MTU.

    <figure>
    <img src="./images/VMware_Network_3=3d1a3a2b-df33-4c36-bb22-af87d7428ca1.webp" />
    </figure>

5.  Click **NEXT**.

6.  On the **Create a Standard Switch** screen, click **+** and select a NIC.

    <figure>
    <img src="./images/VMware_Network_4=5bdb41ae-a93a-46a1-aafb-6119e9f95ec1.webp" /> <img src="./images/VMware_Network_5=e5cde006-b5e7-4605-8ef7-89c2a02d7f0b.webp" />
    </figure>

7.  Click **OK**.

8.  On the **Connection settings** screen, type `Data Network` in the **Network label** field and select **All (4095)** for the **VLAN ID**.

    <figure>
    <img src="./images/VMware_Network_6=f92d7166-d8af-4bd2-9eb2-18eb10162802.webp" />
    </figure>

9.  Click **NEXT**.

10. On the **Ready to complete** screen, review your settings and click **FINISH**.

    <figure>
    <img src="./images/VMware_Network_7=f355ed80-8e42-4ab5-9ac6-f951d0fc0ed9.webp" />
    </figure>

11. Click the ![](/images/VMware_vCenter_server_icon=GUID-20230329175050.webp) tab.

12. Go to **Configure → Networking → Virtual switches**.

13. Find the switch you just created and click **EDIT**.

    <figure>
    <img src="./images/VMware_Network_8=a6933258-aacf-4f4c-ba6c-19e2b6494c0f.webp" />
    </figure>

14. Go to **Security**, and select **Reject** for **Promiscuous mode**, **MAC address changes**, and **Forged transmits**.

    :::note

    Send the network traffic you want to scan to the data port. The following instructions use promiscuous mode to enable Virtual Network Sensor to scan your network traffic.
    :::

    <figure>
    <img src="./images/VMware_Network_9=ad2e3381-b7be-415b-a440-76f8c3294818.webp" />
    </figure>

15. Click **OK**.

16. Click the name of the switch you just created.

17. Click ![](/images/vertical_ellipsis_Data_Network=GUID-20230418113131.webp) next to **Data Network** and select **Edit Settings**.

    <figure>
    <img src="./images/VMware_Network_10=ab0fa5b6-556e-49f2-8a68-18f7d654358d.webp" />
    </figure>

18. Go to **Security**, and select **Override** and **Accept** for **Promiscuous mode**.

19. Select **Reject** for **MAC address changes** and **Forged transmits**, and click **OK**.

    <figure>
    <img src="./images/VMware_Network_11=58c2f8b6-fb3e-4367-baf5-aeff571bbe64.webp" />
    </figure>

    After configuring your network settings, access the Virtual Network Sensor console and use the command `show traffic` to verify the Virtual Network Sensor is receiving traffic. For more information about troubleshooting, see [Virtual Network Sensor FAQ](virtual-network-sensor-faqs.mdx) and [Virtual Network Sensor CLI commands](virtual-network-sensor-cli-commands.mdx).
