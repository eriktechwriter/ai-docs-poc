# Moving Trend Cloud One Agents Complete Guide

Move your agents and custom policies to Trend Vision One Endpoint Security.

The following steps detail how to move agents from Trend Cloud One - Endpoint & Workload Security to the new Trend Vision One Endpoint Security for evaluation of Server & Workload Protection. The steps also include how to import your custom policies and configurations.

### Procedure {#procedure}

1.  In the Trend Cloud One - Endpoint & Workload Security console, go to the **Policies** tab.

2.  Export your customized policies and configurations.

    1.  On the **Policies** tab, select the policy you want to export.

    2.  Click **Export → Export Selected to XML (For Import)**.

    3.  Go to **Common Objects → Lists**.

    4.  Select the custom list you want to export and click **Export Selected to XML (For Import)**.

    5.  Go to **Common Objects → Rules**.

    6.  Select the custom rules you want to export and click **Export Selected to XML (For Import)**.

3.  In the Trend Vision One console, go to **Endpoint Security Operations → Server & Workload Protection**.

    If you have more than one Server & Workload Protection instance, select the instance to import your custom policies and configurations.

4.  Go to the **Policies** tab and import your custom policies and settings.

    1.  On the **Policies** tab, click **New → Import From File**.

    2.  Chose the file to import and select which policy to **Import under**.

    3.  Click **Next** and follow the steps on screen to complete the process.

    4.  Repeat the steps for custom rules and lists in the respective screens.

    5.  After importing, verify all of the custom policies and configurations are available.

5.  If your endpoints require a proxy for external connections, configure the proxy settings in Server & Workload Protection.

    1.  In the Trend Vision One console, go to **Endpoint Security Operations → Server & Workload Protection**.

        If you have more than one Server & Workload Protection instance, select the instance to import your custom policies and configurations.

    2.  Go to **Administration → System Settings → Proxies**.

    3.  Under **Proxy Servers**, click **New → New Proxy Server**.

    4.  Configure the proxy settings and click **OK**.

    5.  Select the proxy you configured for **Primary Component Update Proxy used by Agents, Appliances, and Relays**.

6.  In the Trend Cloud One - Endpoint & Workload Security console, deactivate the agent you want to move.

    - For information on how to deactivate agents, see [Deactivate the agent](https://docs.trendmicro.com/en-us/documentation/article/trend-micro-cloud-one-workload-security-agent-activate) on the Trend Cloud One Help Center.

    - You may also need to disable agent self-protection before you can deactivate the agent. For more information, see [Enable or disable agent self-protection](https://docs.trendmicro.com/en-us/documentation/article/trend-micro-cloud-one-workload-security-agent-self-protection) on the Trend Cloud One Help Center.

7.  In the Trend Vision One console, go to **Endpoint Security Operations → Server & Workload Protection**.

    If you have more than one Server & Workload Protection instance, select the instance where you want to move the agents.

8.  Go to **Administration → Updates → Software → Local**.

9.  Click **Generate Deployment Scripts**.

10. Select the **Platform** and configure the other settings.

11. Copy the activation command from the deployment script.

    The activation command is the line which includes the manager URL, port, `tenantID`, and `token`.

12. Access the device where the target agent is installed.

13. Run Command Prompt as an Administrator.

14. Run the activation command you copied from the deployment script.

15. After reactivating the agent, ensure the correct policies and configurations are applied.
