[Recommendation scans](recommendation-scans-intro.mdx) provide a good starting point for establishing a list of rules that you should implement, but some additional rules for common vulnerabilities are not identified by recommendation scans because must be carefully configured and tested before being implemented in *prevent* (block) mode. Trend Micro recommends that you configure and test these rules, then manually enable them in your policies or individual computers.

The table below includes the most common additional rules you should configure. You can find others in Server & Workload Protection by searching for rules whose type is *Smart* or *Policy*.
