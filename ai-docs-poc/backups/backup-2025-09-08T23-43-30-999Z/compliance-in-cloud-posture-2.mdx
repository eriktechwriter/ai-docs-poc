Trend Vision One™ – Cloud Risk Management provides a number of tools to help organizations quickly assess their infrastructure’s compliance posture against various compliance standards and frameworks:

*Compliance tools:*

### Procedure {#procedure}

1.  Standard and Framework checks report - view how your organization’s infrastructure is tracking against rules filter by various Standards and Frameworks

2.  [Compliance and Cloud Risk Management Reports](compliance-reports.mdx) - view and download a report assessing how your cloud infrastructure security and governance posture is tracking against controls from various Standards and Frameworks.

3.  Compliance Excel Report - Downloadable Excel report of your infrastructure’s compliance posture.

4.  [Compliance Score](compliance-score.mdx) – health metric of your cloud infrastructure measured against Cloud Risk Management’s entire 1,100+ rule set.

## Supported Standards and Frameworks {#supported-standards-and-frameworks}

Cloud Risk Management currently offers reports for the following standards and frameworks. Each standard or framework is made up of controls that specify security and governance requirements. Cloud Risk Management [rules](introduction-rules.mdx) are mapped to these controls and the resulting [checks can be filtered](filter-search.mdx) to display only the rules relevant to a particular standard or framework.

## Standard and Framework checks report {#standard-and-framework-checks-report}

### Procedure {#procedure-1}

1.  Open [All checks report](all-checks-report.mdx)

2.  Select View by *Rule* or by *Resource*

3.  Expand *Filter checks*

4.  Check a standard or framework in *Standards & Frameworks.* For Example: *Monetary Authority of Singapore* TRM

    We currently support the following Standards & Framework filters:

    - AWS Well-Architected Framework
    - Azure Well-Architected Framework
    - Google Cloud Architecture Framework
    - CIS Amazon Web Services Foundations Benchmark v4.0.1
    - CIS Amazon Web Services Foundations Benchmark v5.0.0
    - CIS Microsoft Azure Foundations Benchmark v2.1.0
    - CIS Microsoft Azure Foundations Benchmark v3.0.0
    - CIS Google Cloud Platform Foundation Benchmark v3.0.0
    - CIS Google Cloud Platform Foundation Benchmark v4.0.0
    - CIS Alibaba Foundations Benchmark v1.0.0
    - CIS Alibaba Foundations Benchmark v2.0.0
    - CIS Oracle Cloud Infrastructure Foundation Benchmark v3.0.0
    - CIS Controls Version 8
    - System and Organization Controls (SOC 2) Nov 2019
    - NIST Cybersecurity Framework v1.1
    - NIST Cybersecurity Framework v2.0
    - ISO 27001:2013
    - ISO 27001:2022
    - AusGov ISM Sep 2024
    - HIPAA Feb 2023
    - HITRUST CSF v11.3.0
    - ASAE 3150 Security of CDR Data
    - PCI DSS v3.2.1
    - PCI DSS v4.0.1
    - APRA CPS 234
    - FEDRAMP Rev 4
    - Monetary Authority of Singapore MAS-TRM 2021
    - NIS 2 Directive v2
    - FISC Security Guidelines V12
    - LGPD (Brazil)

5.  Scroll down to the checks list, which will display the standard or framework selected. Click on a rule to see the check result (success or failure) against the rule for each resource. See [Rules](introduction-rules.mdx) for more info.

6.  \[ Optional \] Download the result as a PDF or CSV report.

    - Generate and download new Standard & Framework Checks results
      1.  Click on [Generate report](generate-download-report.mdx)
    - Download previously generated reports from the history
      1.  Expand *Other reports from the* [Configured reports](configured-reports.mdx) list
      2.  Select either CSV or PDF format for the report

    :::note

    Standard and Framework checks reports can also be downloaded from [All Generated Reports](generate-download-report.mdx) list. To know more about standards in a particular Standard and Framework report, in most cases you will need to register with the standard from their website to be able to access a detailed PDF about the standards.
    :::

## CIS Foundations Benchmarks {#cis-foundations-benchmarks}

Not all recommendations from the CIS Benchmarks have been applied. Please refer to the CIS website to access the free [CIS Benchmarks PDFs](https://www.cisecurity.org/cis-benchmarks) for more detail on the recommended settings.

## Compliance Excel Report {#compliance-excel-report}

A Compliance Excel Report is mapped in the same way as a [Compliance & Cloud Risk Management Reports](compliance-reports.mdx) however this report is available in XLS format. Currently only supported for the following CIS Foundations Benchmarks:

- CIS Amazon Web Services Foundations Benchmark
- CIS Microsoft Azure Foundations Benchmark
- CIS Google Cloud Platform Foundation Benchmark
- CIS Alibaba Cloud Foundation Benchmark
- CIS Oracle Cloud Infrastructure Foundation Benchmark
