Cloud Email Gateway Protection provides the following shorthand for writing complicated regular expressions. Cloud Email Gateway Protection will pre-process expressions and translate the shorthand into regular expressions.

For example, {D}+ would be translated to \[0-9\]+. If a shorthand expression is enclosed in brackets (example: {}) or double-quotes, then Cloud Email Gateway Protection will not translate that shorthand expression to a regular expression.

</tr>

<td><p>[0-9]</p></td>

Cloud Email Gateway Protection also provides the following meta-symbols. The difference between shorthand and meta-symbols is that meta-symbols can be within a bracket expression.

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr>
<th><p>META-SYMBOL</p></th>
<th><p>DESCRIPTION</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>\s</p></td>
<td><p>[[:space:]]</p></td>
</tr>
<tr>
<td><p>\S</p></td>
<td><p>[^[:space:]]</p></td>
</tr>
<tr>
<td><p>\d</p></td>
<td><p>[[:digit:]]</p></td>
</tr>
<tr>
<td><p>\D</p></td>
<td><p>[^[:digit:]]</p></td>
</tr>
<tr>
<td><p>\w</p></td>
<td><p>[_[:alnum:]]</p></td>
</tr>
<tr>
<td><p>\W</p></td>
<td><p>[^_[:alnum:]]</p></td>
</tr>
</tbody>
</table>
