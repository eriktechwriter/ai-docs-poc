When Trend Vision One Endpoint Security agents have the mail scan privileges, the **Mail Scan** option displays on the Trend Vision One Endpoint Security agent console. The **Mail Scan** option shows the POP3 mail scan.

The following table describes the POP3 mail scan program.

**Mail Scan Programs**

<table>
<colgroup>
<col style="width: 28%" />
<col style="width: 71%" />
</colgroup>
<thead>
<tr>
<th><p>Details</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Purpose</p></td>
<td><p>Scans POP3 email messages for viruses/malware</p></td>
</tr>
<tr>
<td><p>Prerequisites</p></td>
<td><ul>
<li><p>Must be enabled by administrators from the web console before users can use it</p>


:::note

<p>You must enable the <strong>Display the Mail Scan settings on the Security Agent console</strong> privilege to enable POP3 mail scanning.</p>
<p>For more information, see <a href="trend-vision-one-configure-agent-privileges">Configuring Agent Privileges</a>.</p>


:::

</li>
<li><p>Action against viruses/malware configurable from the Trend Vision One Endpoint Security agent console but not from the web console</p></li>
</ul></td>
</tr>
<tr>
<td><p>Scan types supported</p></td>
<td><p>Real-time Scan</p>
<p>Scanning is done as email messages are retrieved from the POP3 mail server.</p></td>
</tr>
<tr>
<td><p>Scan results</p></td>
<td><ul>
<li><p>Information about detected security risks available after scanning is complete</p></li>
<li><p>Scan results not logged on the Trend Vision One Endpoint Security agent console's <strong>Logs</strong> screen</p></li>
<li><p>Scan results not sent to the server</p></li>
</ul></td>
</tr>
</tbody>
</table>
