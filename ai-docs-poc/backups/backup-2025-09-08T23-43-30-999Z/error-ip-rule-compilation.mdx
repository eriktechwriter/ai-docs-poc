This error can occur for a variety of reasons. To confirm the error is legitimate, resend the policy.

### Procedure {#procedure}

1.  In the Server & Workload Protection console, click **Computers**.

2.  Right-click the computer where the error occurred.

3.  Go to **Actions → Send Policy**.

4.  Re-check the status:

    1.  In the Server & Workload Protection console, click **Computers**.

    2.  Right-click the computer where the error occurred.

    3.  Go to **Actions → Clear Warnings/Errors**.

    4.  Once you have cleared the warnings and errors, go to **Actions → Check Status**.

### Next steps {#next-steps}

If the error continues to occur after completing the previous steps, view the following troubleshooting topics:

- [Apply Intrusion Prevention best practices](apply-best-practices-error.mdx)

- [Manage rules](manage-rules-compilation-error.mdx)

- [Unassign application types from a single port](unassign-application-types-port.mdx)

If the error persists, contact your support provider.
