::: {role="main"}
View information about devices and manage devices on the **Devices** tab.

You can view the following information on the **Devices** tab.

- For integration with Microsoft Endpoint Manager (Intune), VMware Workspace ONE UEM, Google Workspace, or Microsoft Entra ID

  <colgroup>
  <col style="width: 33%" />

  <tr>
  <td><p>Device</p></td>
  <td><p>Device information in the following format: Owner's name + Brand + Model</p>
  <p>Example: <code>John Doe's Apple iPhone 7</code></p>
  <ul>
  <li><p>Click the device name to view detailed information in the device details panel.</p></li>
  </ul></td>
  </tr>
  <tr>
  <td><p>Agent status</p></td>
  <td><p>The status of the Mobile Security for Business app</p>
  <ul>
  <li><p><strong>Normal</strong></p>
  <p>The Mobile Security for Business app is connected to the server and has retrieved the latest policy information.</p></li>
  <li><p><strong>Outdated</strong></p>
  <p>The Mobile Security for Business app is connected to the server and has not retrieved the latest policy information.</p></li>
  <li><p><strong>Inactive</strong></p>
  <p>The Mobile Security for Business app is not connected to the server and has not reported any data for 30 consecutive days.</p></li>
  </ul></td>
  </tr>
  <tr>
  <td><ul>
  <li><p>Operating system</p></li>
  <li><p>OS version</p></li>
  </ul></td>
  <td><p>The operating system running on the device</p>
  <p>When using Intune integration, <strong>Operating system</strong> also includes the enrollment profile type of the device running Android.</p>
  

  :::note
  
  <p>Mobile Security supports the following Android management solution and enrollment profile types:</p>
  <ul>
  <li><p><em>Android Enterprise</em>: <em>Personally-owned devices with work profile</em>, <em>Corporate-owned dedicated devices</em>, and <em>Corporate-owned, fully managed user devices</em></p></li>
  <li><p><em>Android device administrator</em></p></li>
  </ul>
  

  :::

  </td>
  </tr>
  <tr>
  <td><p>Ownership</p></td>
  <td>

:::note

  <p>This attribute is available only when you integrate with Intune or VMware Workspace ONE UEM.</p>
  

  :::

  
  <p>Whether the device is corporate-owned or personally-owned</p></td>
  </tr>
  <tr>
  <td><p>Risk level</p></td>
  <td><p>The risk level that Mobile Security for Business assigns to the device based on the scan result</p>
  <ul>
  <li><p><strong>Low</strong></p>
  <p>Mobile Security detected slightly suspicious characteristics of threats in the last scan. For example, the developer mode or USB debugging is enabled or the operating system is out of date.</p></li>
  <li><p><strong>Medium</strong></p>
  <p>Mobile Security detected moderately suspicious characteristics of threats in the last scan, such as potentially unwanted apps and malicious Wi-Fi networks.</p></li>
  <li><p><strong>High</strong></p>
  <p>Mobile Security detected highly suspicious characteristics of threats in the last scan, such as malicious apps, malicious iOS/iPadOS profile, or rooted device.</p></li>
  </ul></td>
  </tr>
  <tr>
  <td><p>Policy</p></td>
  <td><p>The policy applied to the device</p></td>
  </tr>
  <tr>
  <td><p>Phone Number</p></td>
  <td>

:::note

  <p>This attribute is available only when you integrate with Intune or VMware Workspace ONE UEM.</p>
  

  :::

  
  <p>The phone number of the device</p></td>
  </tr>
  <tr>
  <td><p>IMEI</p></td>
  <td>

:::note

  <p>This attribute is available only when you integrate with Intune or VMware Workspace ONE UEM.</p>
  

  :::

  
  <p>The International Mobile Equipment Identity (IMEI) number of a device</p></td>
  </tr>

  <td><p>User</p></td>
  <td><p>The owner of the device</p></td>
  </tr>

  <td><p>IP address</p></td>
  <td>

:::note

  <p>This attribute is available only when you integrate with Microsoft Entra ID.</p>
  

  :::

  
  <p>The IP address of the device</p></td>
  </tr>
  <tr>

  <td><p>The last time when the device was connected to <strong>Mobile Security</strong></p></td>

You can perform the following actions on the **Devices** tab.

<table>
<colgroup>
<col style="width: 33%" />
<col style="width: 67%" />
</colgroup>
<thead>
<tr>
<th><p>Action</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Filter device data</p></td>
<td><p>Use the following drop-down lists and the search box to locate specific device data:</p>
<ul>
<li><p><strong>Agent status</strong></p>


:::note

<p>This filter is not available when you integrate with an MDM through managed configuration</p>


:::

</li>
<li><p><strong>Operating system</strong></p></li>
<li><p><strong>Risk level</strong></p></li>
<li><p><strong>Last connected</strong></p></li>
</ul></td>
</tr>
<tr>
<td><p>Unenroll devices</p></td>
<td><p>Select one or more devices from the list and click <strong>Unenroll Device</strong>.</p>


:::note

<p>Trend Micro recommends unenrolling devices that have been transferred to other users or made obsolete.</p>


:::

</td>
</tr>
<tr>
<td><p>View device details</p></td>
<td><p>Click a device name. View information about the device hardware, operating system, and security profile. You can also click the links to access information about the most recent detections.</p>
<p>If a device's risk level is <strong>Medium</strong> or <strong>High</strong>, you can click <img src="./images/Right_arrow_icon=GUID-7E5A9DB2-5872-4CB7-BF57-C8FD892D60DB=1=en-us=Low.webp" /> to view the scan result of the device.</p></td>
</tr>
<tr>
<td><p>Wipe devices</p></td>
<td><p>Click the Wipe action icon (<img src="./images/wipe_device=20230317090707.webp" />) corresponding to a device.</p>


:::note

<p>This action applies to Android devices using Microsoft Intune and VMware Workspace ONE UEM integration only.</p>


:::


<p>The Wipe action restores devices to their factory defaults, permanently deletes all data from the devices, and removes the devices from the Mobile Inventory list.</p>


:::note

<p>Mobile Security cannot remotely wipe Android devices with the <em>Personally-owned devices with work profile</em> type.</p>


:::

</td>
</tr>
<tr>
<td><p>Lock devices</p></td>
<td><p>Click the Remote Lock action icon (<img src="./images/remote_lock_device=dea03be0-cd7e-4a62-a611-282000b0a931.webp" />) corresponding to a device.</p>


:::note

<p>This action applies to Android devices using Microsoft Intune and VMware Workspace ONE UEM integration only.</p>


:::


<p>The Remote Lock action locks a device. To unlock the device, the device owner needs to enter their passcode.</p>


:::note

<p>Mobile Security cannot remotely wipe Android devices that do not have a PIN or password configured.</p>


:::

</td>
</tr>
<tr>
<td><p>Delete devices</p></td>
<td><p>Click the delete icon (<img src="./images/delete_policy=cce60f96-0e5c-4565-895f-d648c42e8495.webp" />) corresponding to a device or select one or more groups from the list and click <strong>Delete</strong>.</p></td>
</tr>
<tr>
<td><p>Start a manual scan</p></td>
<td><p>Click the Start Manual Scan icon (<img src="./images/manual_scan_icon=20250212050123.webp" />) corresponding to a device.</p></td>
</tr>
</tbody>
</table>
