View information about the **System configuration** risk factor, which is determined by misconfigured security settings in your managed assets.

The System Configuration risk factor contributes to your [exposure level](exposure-overview.mdx).

The following table outlines the widgets available in the System Configuration section.

<tr>
<td><p>Risk Metrics</p></td>
<td><p>View information about specific configuration risks</p>
<ul>
<li><p>The <strong>Internet-Facing asset configuration</strong> section displays the number of hosts with insecure connection or unexpected internet-facing services or ports.</p></li>
<li><p>The <strong>Cloud infrastructure configuration</strong> section displays high-risk configurations and <a href="trend-vision-one-cloud-asset-compliance-violations">compliance violations</a> effecting your cloud infrastructure.</p></li>
<li><p>The <strong>IAM configuration</strong> section displays <a href="trend-vision-one-accounts-weak-authentication">accounts with weak authentication</a>, <a href="trend-vision-one-accounts-excessive-privilege">excessive privilege</a>, sign-on activity from legacy authentication protocols, and settings that <a href="trend-vision-one-accounts-attack-surface-risk">increase attack surface risk</a>.</p></li>

When viewing risk events, click the number in the case column to view current cases involving the specified risk event. Click the options icon (![](/images/options=ddb0b67f-0654-4aa5-8bc7-48ec554c5448.webp)) to open a new case for the risk event or add the case to an existing risk event.

You can also select up to five internet-facing subdomains or public IPv4 addresses to scan for exposures. For more information, see [Internet-facing asset exposure scans](internet-facing-asset-scan.mdx).
