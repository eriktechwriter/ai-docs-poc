::: {role="main"}
The **Intrusion Prevention Rule Properties** screen displays detailed information about a specific Intrusion Prevention Rule and vulnerability. Click the **General** tab or the **Vulnerability** to view details about the rule.

The following tables describe the information provided on the **General** tab and **Vulnerability** tab.

**General Information**

<table>
<colgroup>
<col style="width: 25%" />
<col style="width: 75%" />
</colgroup>
<thead>
<tr>
<th><p>Data</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Identifier</p></td>
<td><p>The unique identifier tag for the Intrusion Prevention Rule</p></td>
</tr>
<tr>
<td><p>Name</p></td>
<td><p>The name of the Intrusion Prevention Rule</p></td>
</tr>
<tr>
<td><p>Description</p></td>
<td><p>The description of the Intrusion Prevention Rule</p>


:::note

<p>Apex One Vulnerability Protection does not support the configuration options available on the standalone version of Trend Micro Vulnerability Protection.</p>


:::

</td>
</tr>
<tr>
<td><p>Application Type</p></td>
<td><p>The Application Type this Intrusion Prevention Rule is grouped under</p></td>
</tr>
<tr>
<td><p>Priority</p></td>
<td><p>The priority level of the Intrusion Prevention Rule. Higher priority rules are applied before lower priority rules.</p></td>
</tr>
<tr>
<td><p>Severity</p></td>
<td><p>The severity level that Trend Micro assigns to the rule</p>


:::note

<p>The severity of a rule has no effect on how the rule is implemented or applied. Severity levels can be useful as sorting criteria when viewing a list of Intrusion Prevention Rules.</p>


:::

</td>
</tr>
<tr>
<td><p>Mode</p></td>
<td><p>The network engine detection mode used by the Intrusion Prevention module. Click a mode to configure the setting for the rule.</p></td>
</tr>
<tr>
<td><p>Type</p></td>
<td><p>The type of vulnerability detected:</p>
<ul>
<li><p><em>Smart</em>: Known or unknown (for example, zero-day) vulnerability</p></li>
<li><p><em>Exploit</em>: Known exploit (usually signature based) for a known vulnerability</p></li>
<li><p><em>Vulnerability</em>: Known vulnerability for which one or more exploits may exist</p></li>
</ul></td>
</tr>
<tr>
<td><p>Issued</p></td>
<td><p>The date the rule was released (not downloaded)</p></td>
</tr>
<tr>
<td><p>Last Updated</p></td>
<td><p>The date and time the rule was last modified</p></td>
</tr>
</tbody>
</table>
