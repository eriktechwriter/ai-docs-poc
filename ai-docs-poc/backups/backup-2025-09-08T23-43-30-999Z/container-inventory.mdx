:::: {role="main"}
Connect Kubernetes and Amazon ECS clusters, deploy Container Protection policies and proxy settings, create and manage Kubernetes cluster groups, and monitor your container environment from Container Inventory.

Different options are available for Kubernetes and Amazon ECS clusters. The options are outlined in the tables below.

:::tip

- You can find assets by using the search function or clicking the filter icon (![](/images/filter_icon=GUID-1d5c17ad-8efa-46f7-a1c7-33cbc1b1329c.webp)) to apply filters by column headings.
- You can create, manage, and assign tags to any cluster by clicking **Add filter â†’ Tags**. For more information on tags, see [Tag Management](tag-management.mdx).
:::

### Amazon ECS {#amazon-ecs}

The following table outlines the options available for Amazon ECS cluster.

<table>
<colgroup>
<col style="width: 25%" />
<col style="width: 75%" />
</colgroup>
<thead>
<tr>
<th><p>Option</p></th>
<th><p>Options</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>View details about your container environment</p></td>
<td><p>Container Inventory provides a tree view to manage the protection of all your connected Amazon ECS clusters.</p>
<p>Amazon ECS hierarchy: Orchestration platform (Amazon ECS) &gt; Cloud account &gt; Region &gt; Cluster &gt; Services &gt; Tasks &gt; Containers</p></td>
</tr>
<tr>
<td><p>Add new Amazon ECS clusters</p></td>
<td><p>Select <strong>Amazon ECS</strong>, a cloud account, or a region from the tree. Click <strong>Add account</strong> (if other clusters have previously been added) or <strong>Add and protect Amazon ECS assets</strong> (if no clusters have previously been added).</p>
<p>For detailed instructions, see:</p>
<ul>
<li><p><a href="trend-vision-one-ecs-clusters-new-aws-account">Connect Amazon ECS clusters using a new AWS account</a></p></li>
<li><p><a href="trend-vision-one-fargate-clusters-amazon-ecs">Set up connected Amazon ECS Fargate clusters</a></p></li>
</ul></td>
</tr>
<tr>
<td><p>Enable or disable runtime security and scanning on clusters</p></td>
<td><p>Select one or more clusters from the tree to enable or disable runtime security or runtime scanning.</p>
<ul>
<li><p><strong>Runtime security</strong>: Provides visibility into any activity of your running containers that violates a customizable set of rules.</p></li>
<li><p><strong>Runtime scanning</strong>: Provides visibility of operating system and open source code vulnerabilities that are part of containers running in clusters.</p></li>
</ul></td>
</tr>
<tr>
<td><p>Assign new policies</p></td>
<td><p>Select a cluster from the tree. In the <strong>Policy</strong> field, select from existing Container Protection policies or click <strong>View Policy editor in Container Protection App</strong> to modify the current policy settings.</p></td>
</tr>
</tbody>
</table>
