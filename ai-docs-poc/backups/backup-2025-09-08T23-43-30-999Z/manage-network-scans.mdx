:::: {role="main"}
View, edit, trigger, and see results for your configured network scans.

:::warning[Important]

This is a "Pre-release" feature and is not considered an official release. Please review the [Pre-release disclaimer](pre-release-disclaimer.mdx) before using the feature.
:::

The **Network scans** tab displays your currently configured scans along with the scan status, details, and trigger method. The following table details the actions available in **Network scans**.

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr>
<th><p>Action</p></th>
<th><p>Details</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Trigger a scan</p></td>
<td><p>Click the run icon (<img src="./images/trash_icon=GUID-47cf6867-6315-438e-8670-86ff36f22a28.webp" />) to manually trigger a scan. Scans already in progress during the scheduled scan time will take place at the next scheduled time.</p></td>
</tr>
<tr>
<td><p>View and edit scan details</p></td>
<td><p>Click the name of a scan to view the scan details and edit the configuration.</p></td>
</tr>
<tr>
<td><p>View last scan report</p></td>
<td><p>Click <strong>View</strong> in the <strong>Last scan report</strong> column to see details of the last scan report in <strong>Scan reports</strong>. Only the most recent scan report for the scan is available.</p></td>
</tr>
<tr>
<td><p>Change scan status</p></td>
<td><p>Change the scan status to enabled or disabled in the <strong>Status</strong> column. Disabling a scan does not delete any previously generated scan reports.</p></td>
</tr>
<tr>
<td><p>Delete a scan</p></td>
<td><p>Click the delete icon (<img src="./images/trash_icon=GUID-47cf6867-6315-438e-8670-86ff36f22a28.webp" />) to delete the scan. Deleting a scan does not delete any previously generated scan reports. The last report for a deleted scan is available in <strong>Scan Reports</strong>.</p></td>
</tr>
</tbody>
</table>
