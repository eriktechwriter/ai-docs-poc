Configure the connector to enable sharing Trend Vision One XDR data with Splunk Cloud.

The Splunk HEC connector utilizes the HTTP Event Collector to send XDR data to Splunk Cloud. The connector supports connections to multiple Splunk Cloud instances.

### Procedure {#procedure}

1.  In Trend Vision One, go to **Workflow and Automation â†’ Third-Party Integration**.

2.  Click **Splunk HEC Connector (SaaS/Cloud)**.

3.  Click **+ Connect Splunk HEC Server**.

    The **Splunk HEC Server Connection** window appears.

4.  Configure the connection settings in the **Splunk HEC Server Connection** panel.

5.  Configure the scope of data to send to Splunk Cloud by selecting from the following:

    - Workbench alerts

    - Events

      - Observed Attack Techniques (requires specifying event severity)

      - All detections

    - Container vulnerabilities

      :::note

      Due to the potentially large size of container vulnerability data, Trend Micro recommends changing the Splunk TRUNCATE setting from the default of 10,000 to 100,000. For more information about the TRUNCATE setting, check the Splunk official documentation.
      :::

    - Activity data (requires specifying scope)

      :::note

      Sending activity data requires Trend Vision One credits. [Configure the data allowance for transferring activity data and manage credit allocation](credit-req-for-apps-services.mdx) in the **Credits & Billing** app.
      :::

    For example, you can configure the following in Splunk:

    TIME_PREFIX = ("eventTime":\s\*)|("scantime":\s\*)|("updatedTime":\s\*)|("detectionTime":\s\*)

    For further Splunk questions, please consult your Splunk support.

6.  Select the assets from which the specified data is collected.

    - All assets: Data from all assets is transferred

    - Assets with selected tags: Only data from assets with the specified tags is transferred

      :::note

      Currently, only data from endpoints and container clusters tagged with the selected tags can be transferred to Splunk Cloud.

      You can select a maximum of 20 tags.
      :::

7.  Click **Test Connection** to verify if the settings are valid.

8.  Click **Connect**.

    The Splunk HEC server appears on the **Splunk HEC Connector (SaaS/Cloud)** screen.

9.  You can repeat the previous steps to add multiple Splunk HEC servers with their own data source configurations.

10. You can use the ![](/images/edit_icon=GUID-1F1D1164-5310-4D6D-ACD0-6049C86960AF.webp) or ![](/images/trash_icon=GUID-47cf6867-6315-438e-8670-86ff36f22a28.webp) icons to modify or delete a server from the **Splunk HEC Connector (SaaS/Cloud)** screen.
