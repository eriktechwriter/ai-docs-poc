Gain insights into activities regarding access to your corporate apps from your users and devices.

The following table outlines the available widgets.

<col style="width: 25%" />
<col style="width: 75%" />

<th><p>Widget</p></th>
<th><p>Description</p></th>
</tr>

<tr>
<td><p>Total Private Access Attempts</p></td>
<td><ul>
<li><p>Total Attempts: The number of attempts to access internal apps during the selected time period</p></li>
<li><p>Blocked attempts: The number of attempts to access internal apps that were blocked during the selected time period</p></li>
</ul></td>
</tr>
<tr>
<td><p>Users</p></td>
<td><ul>
<li><p>Total Users: The number of users who attempted to access internal apps during the selected time period</p></li>
<li><p>Blocked Users: The number of users who attempted to access internal apps and were blocked during the selected time period</p></li>
<li><p>Active Connections: The number of users who are currently accessing internal apps</p>
<p>Click <img src="./images/download_icon=5c7476c2-cf15-4572-b7cd-5fc67a57d22f.webp" /> to export the user data into a CSV file.</p></li>
</ul></td>
</tr>
<tr>
<td><p>Devices</p></td>
<td><ul>
<li><p>Total Devices: The number of devices that were used to access internal apps during the selected time period</p></li>
<li><p>Blocked Devices: The number of devices that were used to access internal apps and were blocked during the selected time period</p></li>
<li><p>Active Connections: The number of devices that are currently accessing internal apps</p>
<p>Click <img src="./images/download_icon=5c7476c2-cf15-4572-b7cd-5fc67a57d22f.webp" /> to export the device data into a CSV file.</p></li>
</ul></td>
</tr>
<tr>
<td><p>Internal apps</p></td>
<td><ul>
<li><p>Total Apps: The number of internal apps that users and devices attempted to access during the selected time period</p></li>
<li><p>Blocked Apps: The number of internal apps with blocked attempts during the selected time period</p></li>
</ul></td>
</tr>
<tr>
<td><p>User Access Overview</p></td>
<td><ul>
<li><p>Top Users by Blocked Attempts: The top 10 users whose private access attempts were blocked the most</p></li>
<li><p>Top Users by Monitored/Blocked Attempts: The top 10 users whose private access attempts were monitored or blocked the most</p></li>
<li><p>User Sign-in History: The users who have signed in for Private Access, either through the Secure Access Module or the browser-based user portal</p>
<p>Information includes the last device that the user used to sign in, the number of devices that the user has used to sign in, and the last sign-in time.</p></li>
</ul>
<p>To view activity logs for an entry, click the corresponding block count or access count.</p>
<p>To gain insights into access activities from more users, synchronize more user data by clicking <strong>Configure IAM</strong>.</p></td>
</tr>
<tr>
<td><p>Device Access Overview</p></td>
<td><ul>
<li><p>Top Devices by Blocked Attempts: The top 10 devices, by name or IP address, whose private access attempts were blocked the most</p></li>
<li><p>Top Devices by Monitored/Blocked Attempts: The top 10 devices, by name or IP address, whose private access attempts were monitored or blocked the most</p></li>
<li><p>Device Sign-in History: The devices that have been used to sign in for Private Access, either through the Secure Access Module or the browser-based user portal</p>
<p>Information includes the last user who signed in on the device, the number of users who have used the device to sign in, and the last sign-in time.</p></li>
</ul>
<p>To view activity logs for an entry, click the corresponding block count or access count.</p>
<p>To gain insights into access activities from more devices, deploy the Secure Access Module to more devices by clicking <strong>Deploy Secure Access Module</strong>.</p></td>
</tr>
<tr>
<td><p>Internal App Access Overview</p></td>
<td><ul>
<li><p>Top Internal Apps by Blocked Attempts: The top 10 internal apps with the most blocked access attempts</p></li>
<li><p>Top Internal Apps by Monitored/Blocked Attempts: The top 10 internal apps with the most monitored or blocked access attempts</p></li>
</ul>
<p>To view activity logs for an entry, click the corresponding block count or access count.</p>
<p>To gain insights into more internal app access activities, add more internal apps by clicking <strong>Configure Applications</strong>.</p></td>
</tr>
