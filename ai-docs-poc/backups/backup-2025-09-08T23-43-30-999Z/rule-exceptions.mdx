You can configure the Rule Exceptions list in Trend Vision One Endpoint Security agents policies to exclude detections based on rule IDs from the following scan features:

- Anti-malware scans

- Behavior Monitoring

- Suspicious Connection

Configure rule exceptions in Trend Vision One Endpoint Security agents policies to enhance policy management and improve scan performance.

The following table lists the actions you can perform on the **Rule Exceptions** screen.

<table>
<colgroup>
<col style="width: 25%" />
<col style="width: 75%" />
</colgroup>
<thead>
<tr>
<th><p>Action</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Add a rule ID</p></td>
<td><p>Type a rule ID and click <strong>Add</strong>.</p>
<p>You can find rule IDs on the <strong>Log Query</strong> screen.</p>
<p>For more information, see <a href="trend-vision-one-configuring-rule-exceptions">Configuring Rule Exceptions</a>.</p></td>
</tr>
<tr>
<td><p>Delete a rule ID</p></td>
<td><p>Click the <strong>Delete</strong> icon to remove a rule ID from the list.</p></td>
</tr>
<tr>
<td><p>Import rule exceptions</p></td>
<td><p>Click <strong>Import</strong> and select a CSV file containing rule IDs.</p></td>
</tr>
<tr>
<td><p>Export rule exceptions</p></td>
<td><p>Click <strong>Export</strong> to export the Rule Exceptions list to a CSV file.</p></td>
</tr>
</tbody>
</table>
