::: {role="main"}
View the permissions currently granted to registered applications in your organization to better understand the applications authorized to use a resource.

The **Granted Permission** tab allows you to view the permissions granted in your organization to registered applications. Users grant applications permission to access or modify resources.

The following table details the columns on the **Granted Permission** tab.

<table>
<colgroup>
<col style="width: 25%" />
<col style="width: 75%" />
</colgroup>
<thead>
<tr>
<th><p>Column name</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Permission name</p></td>
<td><p>Name of a permission granted to applications.</p></td>
</tr>
<tr>
<td><p>Permission type</p></td>
<td><p>Type of a permission.</p>
<ul>
<li><p>Delegated permissions: Allows an application to access a resource on behalf of a signed-in user.</p></li>
<li><p>Application permissions: Allows an application to access a resource without a signed-in user present.</p></li>
</ul></td>
</tr>
<tr>
<td><p>Authorized resource</p></td>
<td><p>API that an application is granted a permission to call.</p></td>
</tr>
<tr>
<td><p>Authorized applications</p></td>
<td><p>Applications that are granted a permission.</p>
<p>Click the number to view a list of applications to which the users have granted the permission.</p></td>
</tr>
<tr>
<td><p>Authorizing users</p></td>
<td><p>Users granting a permission to applications.</p>
<p>Click the number to view a list of users who have granted the permission to the applications.</p></td>
</tr>
<tr>
<td><p>Privileged</p></td>
<td><p>Indicates whether a permission is a privileged permission.</p>
<p>Privileged permissions can be used to delegate directory resource management to other users, modify credentials, modify authentication or authorization policies, or access restricted data. For details about privileged permissions, see <a href="https://learn.microsoft.com/en-us/entra/identity/role-based-access-control/permissions-reference">Microsoft's documentation</a>.</p></td>
</tr>
</tbody>
</table>
