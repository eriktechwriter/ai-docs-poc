:::: {role="main"}
Learn about alternative methods that ensure internet traffic is directed to the Internet Access Gateway without installing the Secure Access Module.

:::note

You can use PAC files and proxy chaining concurrently.
:::

<td><p>Proxy auto-configuration (PAC) files</p></td>
<td><p>When your organization already has a proxy configured and you support users that often work outside the network</p>

<li><p>Works on internal and external networks</p></li>
</ul></td>
<td><ul>
<li><p>Users with admin rights can bypass using non-standard browsers</p></li>
<li><p>Users can bypass by manually changing to "no proxy"</p></li>
</ul></td>

<td><p>Proxy chaining</p></td>
<td><p>When your organization already has a proxy configured and you do not need to support users that often work outside the network</p></td>
<td><ul>
<li><p>Easy to set up</p></li>
<li><p>Multiple rules offer full redundancy</p></li>
<li><p>Supported by most web proxies</p></li>
<li><p>Users cannot bypass</p></li>
</ul></td>
<td><ul>
<li><p>Only internal network users are protected</p></li>
</ul></td>
</tr>
<tr>
<td><p>Port forwarding</p></td>
<td><p>Not recommended due to the manual configuration involved</p></td>
<td><ul>
<li><p>Easy to set up</p></li>
<li><p>Supported by major browsers</p></li>
<li><p>Users cannot bypass</p></li>
<li><p>Supports HTTP and HTTPS traffic</p></li>
</ul></td>
<td><ul>
<li><p>Requires manual changes if primary gateway is unavailable</p></li>
<li><p>Only internal network users are protected</p></li>
</ul></td>
