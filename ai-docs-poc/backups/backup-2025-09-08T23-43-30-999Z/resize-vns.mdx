Learn how to resize the Virtual Network Sensor according to the needs of your environment.

The following steps show how to resize a Virtual Network Sensor that is deployed on a hypervisor. If your Virtual Network Sensor is deployed on a cloud platform, see the following documentation for instructions on resizing:

- [Amazon Web Services](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-instance-resize.mdx)

- [Microsoft Azure](https://learn.microsoft.com/en-us/azure/virtual-machines/sizes/resize-vm)

- [Google Cloud](https://cloud.google.com/compute/docs/instances/changing-machine-type-of-stopped-instance)

### Procedure {#procedure}

1.  On the hypervisor console, turn off the Virtual Network Sensor.

2.  Change the CPU, memory, and disk size according to your requirements.

    :::note

    Disk size can only be increased.
    :::

3.  Save your settings.

4.  Turn on the Virtual Network Sensor.

5.  Go to **Network Security → Network Inventory → Virtual Network Sensor** to view the updated configured bandwidth.
:::
