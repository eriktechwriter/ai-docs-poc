Move selected Trend Cloud One Endpoint & Workload Security agents to Server & Workload Protection to experience the new Trend Vision One Endpoint Security.

Before you move any agents to evaluate the new Trend Vision One Endpoint Security, ensure you have completed the following:

1.  Access the Foundation Services Update of the Trend Vision One console.

    - For new users, [activate your Trend Vision One console](accessing-your-console.mdx).

    - For current Trend Vision One users, [upgrade your console to the latest version](update-foundation-services-release.mdx).

2.  [Create a Server & Workload Protection instance](creating-new-product-instance.mdx) in Product Instance.

    :::note

    You can provision up to five Server & Workload Protection Manager instances.
    :::

3.  [Deploy a Service Gateway and configure your firewall settings](deploy-service-gateway-exceptions.mdx).

Once you are ready, use the following guides to move your Trend Cloud One - Endpoint & Workload Security agents:

- To only move agents without importing custom settings, see [Moving Trend Cloud One Agents Quick Guide](move-dsa-quick-guide.mdx).

- To move agents and import custom settings, see [Moving Trend Cloud One Agents Complete Guide](move-dsa-complete-guide.mdx).

If you need to move an agent back to Trend Cloud One, see [Returning Agents to Trend Cloud One - Endpoint & Workload Security](return-dsa-cloud-one.mdx).

Once you are ready to update Trend Cloud One - Endpoint & Workload Security to Trend Vision One Endpoint Security, see [Update from Trend Cloud One - Endpoint & Workload Security](update-workload-security.mdx)
