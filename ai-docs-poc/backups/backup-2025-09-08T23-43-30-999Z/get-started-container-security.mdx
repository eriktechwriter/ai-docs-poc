Get Container Security protection up and running quickly and start monitoring your container environment for vulnerabilities and policy violations in 5 easy steps.

1.  Connect a cluster:

    - [Connect an Amazon EKS cluster](eks-clusters-without-fargate.mdx)

    - [Connect a Microsoft AKS cluster](connecting-microsoft-aks-clusters.mdx)

    - [Connect a Google GKE cluster](connecting-google-gke-clusters.mdx)

    - [Connect Alibaba Cloud ACK clusters](connect-alibaba-ack-clusters.mdx)

2.  [Create a Runtime Scanning ruleset](runtime-security-ruleset.mdx)

3.  [Create a Container Protection policy](create-container-policy.mdx)

4.  Deploy Container Security to one or more of your Kubernetes or Amazon ECS clusters.

    - Kubernetes: [Amazon EKS clusters (with and without Fargate)](eks-clusters-without-fargate.mdx), [Microsoft AKS clusters](connecting-microsoft-aks-clusters.mdx), [Google GKE clusters](connecting-google-gke-clusters.mdx), [Connect Alibaba Cloud ACK clusters](connect-alibaba-ack-clusters.mdx)

    - Amazon ECS: [Amazon ECS clusters](ecs-clusters-new-aws.mdx) or [Amazon ECS Fargate clusters](amazon-ecs-fargate-clusters.mdx)

5.  Monitor [policy violations](events.mdx) and [vulnerability detections](vulnerabilities-container-prot.mdx)

:::note

Container Security supports asset configuration and management with Terraform. For details, see the [Trend Vision One Terraform Provider](https://registry.terraform.io/providers/trendmicro/vision-one/latest).
:::
