Some Trend Micro Email Security features have changed behavior or limited support in Cloud Email Gateway Protection.

<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr>
<th><p>Trend Micro Email Security Feature</p></th>
<th><p>Change in Cloud Email Gateway Protection</p></th>
</tr>
</thead>
<tbody>
<tr>
<td><p>Audit logs</p></td>
<td><p>Log query permissions:</p>
<ul>
<li><p>Root account: All created logs</p></li>
<li><p>Accounts with Service Management permissions: Logs not created by the root account</p></li>
<li><p>Accounts without Service Management permissions: Only logs created by the account holder</p></li>
<li><p>Cannot query for activity performed by a sub-account</p></li>
</ul></td>
</tr>
<tr>
<td><p>API keys</p></td>
<td><p>API key management:</p>
<ul>
<li><p>Local accounts upgraded from Trend Micro Email Security: Permitted</p></li>
<li><p>Trend Vision One local accounts: Not permitted</p></li>
</ul></td>
</tr>
<tr>
<td><p>Logon access control</p></td>
<td><p>Console access control:</p>
<ul>
<li><p>Trend Vision One console access is managed in <strong>Console Settings</strong></p></li>
<li><p>End user console access is managed by Cloud Email Gateway Protection.</p></li>
</ul></td>
</tr>
<tr>
<td><p>Co-Branding</p></td>
<td><p>Co-branding management:</p>
<ul>
<li><p>Only end-user console co-branding is supported</p></li>
<li><p>Co-branding the Trend Vision One console is not supported</p></li>
</ul></td>
</tr>
<tr>
<td><p>Account provisioning</p></td>
<td><p>Managed in Trend Vision One <strong>User Accounts</strong>; profile creation no longer required</p></td>
</tr>
<tr>
<td><p>Encrypted email</p></td>
<td><p>Requires additional credit allocation. Not available in all regions.</p></td>
</tr>
</tbody>
</table>
